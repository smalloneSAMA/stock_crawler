> 基于 `stock_crawler` 源码全量通读，梳理本工具对 A 股股票可进行的**所有分析维度**，以及如何用这些分析指导买卖决策。

## 一、数据底座：我们能拿到什么？

### 1.1 行情数据（日K线）

| 字段                                     | 来源                                 | 说明                                 |
| ---------------------------------------- | ------------------------------------ | ------------------------------------ |
| 日期 / 开盘价 / 最高价 / 最低价 / 收盘价 | mootdx(通达信) > 新浪财经 > 腾讯财经 | 三级级联降级，自动分页获取全量历史   |
| 成交量(万手)                             | 同上                                 | 自动单位归一（股→万手）             |
| 成交额(亿)                               | 新浪实时行情 + 均价估算              | 最新一日用实时行情精确修正           |
| 涨跌幅%                                  | 本地计算                             | (今收-昨收)/昨收 × 100              |
| 振幅%                                    | 本地计算                             | (最高-最低)/昨收 × 100              |
| 日内波动区间%                            | 本地计算                             | (最高-最低)/开盘 × 100              |
| 上涨幅度% / 下跌幅度%                    | 本地计算                             | (最高-开盘)/开盘 和 (最低-开盘)/开盘 |

### 1.2 财务数据（逐行填充到每根K线）

| 字段               | 来源              | 说明                  |
| ------------------ | ----------------- | --------------------- |
| 当前市值(亿)       | 东财财报 + 收盘价 | 总股本 × 收盘价      |
| 市盈率TTM          | 同上              | 用TTM每股收益滚动计算 |
| 市净率             | 同上              | 收盘价/每股净资产     |
| 市销率             | 同上              | 市值/TTM营业收入      |
| 净资产收益率%(ROE) | 同上              | 最近完整财年的ROE     |
| 每股收益(TTM)      | 同上              | year-to-date 滚动计算 |
| 每股股息TTM        | 东财财报推算      | EPS - Δ未分配利润    |
| 股息率TTM          | 每股股息/收盘价   | 逐日变化              |
| 分红率             | 同财年股息/EPS    | 整年统一值            |

## 二、四大分析引擎

### 🔍 引擎 1：趋势选时（`analyze_trend()`）

**回答的核心问题：** 现在该不该动手？趋势是多是空？

**评分体系（满分100分）：**

| 维度     | 权重 | 分析内容                                |
| -------- | ---- | --------------------------------------- |
| 均线位置 | 50分 | 价格是否站上 MA5/MA10/MA20/MA30/MA60    |
| 均线排列 | 20分 | 短>中>长 = 多头排列 加分                |
| MACD动量 | 15分 | 金叉/死叉/柱体方向                      |
| RSI动能  | 10分 | 超买(>80)/偏强(>60)/偏弱(<40)/超卖(<20) |
| 成交量   | 5分  | 放量/缩量配合                           |

**回调计算：** 价格偏离各均线的乖离率（%）

**结论：**

- ≥70 分 → **[加仓]** 趋势偏多，可择机加仓
- 50~69 分 → **[持仓观望]** 趋势中性，等待方向
- 30~49 分 → **[谨慎减仓]** 趋势偏弱，逢反弹减仓
- <30 分 → **[减仓/离场]** 趋势明显弱势

**实战指导：**

- 趋势评分持续走低 → 不在下跌趋势中加仓（不接飞刀）
- 趋势评分从 <30 反弹到 >50 → 可能是趋势反转信号，关注买点
- MA60 是牛熊分界线，价格在 MA60 之下不宜重仓

---

### 💰 引擎 2：估值定仓（`analyze_valuation()`）

**回答的核心问题：** 如果动手，仓位给多少？

**评分体系（满分100，含4个子维度）：**

| 子维度      | 权重 | 计算方式                         |
| ----------- | ---- | -------------------------------- |
| PE分位评分  | 25%  | 100 - PE历史百分位（越低分越高） |
| PB分位评分  | 25%  | 100 - PB历史百分位（越低分越高） |
| ROE质量评分 | 35%  | 近1年均ROE × 4（越高分越高）    |
| 股息评分    | 15%  | 股息率 × 20（5%股息率=满分）    |

**仓位建议：**

| 评分  | 建议仓位      | 说明                               |
| ----- | ------------- | ---------------------------------- |
| ≥80  | 90%（重仓）   | 低估区间 + 优质ROE，适合大仓位布局 |
| 65~79 | 70%（中高仓） | 偏低估值，中等偏上仓位             |
| 45~64 | 50%（半仓）   | 合理估值区间，半仓操作             |
| 25~44 | 30%（低仓）   | 偏高估值，轻仓参与                 |
| <25   | 10%（观察仓） | 高估区间，仅观察仓                 |

**输出：**

- PE/PB 的历史 **最低/中位/最高/当前百分位** + 可视化进度条
- ROE 质量评价：优秀(>20%) / 良好(15-20%) / 一般(10-15%) / 较差(<10%)
- 估值温度计：过热 / 偏高 / 合理 / 偏低 / 低估

**实战指导：**

- PE百分位 < 30% 且 ROE > 15% = 理想的买入区间
- PE百分位 > 70% 且 ROE < 10% = 高估低质，应该减仓或清仓
- 高ROE + 低PE = "又好又便宜"，是价值投资最理想的标的

---

### 📐 引擎 3：波动降本（`analyze_volatility()`）

**回答的核心问题：** 怎么用做T和网格降低持仓成本？

#### 3.1 波动特征

| 指标          | 说明                                                             |
| ------------- | ---------------------------------------------------------------- |
| ATR(14)       | 平均真实波幅，日内波动的核心量化指标                             |
| ATR/股价%     | ATR 占股价百分比，衡量波动剧烈程度                               |
| 近20日均振幅% | 日内高低点差的均值                                               |
| 振幅标准差    | 波动稳定性指标                                                   |
| 波动评级      | 高波动(>4%)适合做T / 中波动(>2.5%)适合网格 / 低波动(<2.5%)需耐心 |

#### 3.2 关键价位（支撑/阻力）

基于60日最高/最低 + ATR 动态计算 3 级支撑和 3 级阻力：

```
阻力3(强) ─── 阻力2 ─── 阻力1(近)
                            ← 当前价
支撑1(近) ─── 支撑2 ─── 支撑3(强)
```

#### 3.3 日内做T策略

| 要素      | 计算方式                                           |
| --------- | -------------------------------------------------- |
| 买入区间  | 当前价 - ATR×0.5 ~ 当前价 - ATR×0.2              |
| 卖出区间  | 当前价 + ATR×0.2 ~ 当前价 + ATR×0.5              |
| 单次T利润 | ATR × 0.4 元/股                                   |
| 单笔数量  | 根据近20日均成交额自动估算（每笔约成交额的1/1000） |

#### 3.4 多日波段T网格

基于 ATR × atr_multiplier 计算网格间距（默认 0.5 × ATR）：

| 要素     | 说明                           |
| -------- | ------------------------------ |
| 买入N档  | 当前价 - 网格间距 × N         |
| 卖出N档  | 当前价 + 网格间距 × N         |
| 每档资金 | 单笔数量 × 当前价             |
| 满格利润 | 网格间距 × 单笔数量 × 档位数 |

#### 3.5 T梯度统计（基于全量历史数据）

对上涨幅度%和下跌幅度%按 1% 梯度统计：

| 梯度 | 正T胜率 | 正T天数 | 反T胜率 | 反T天数 | 推荐策略 |
| ---- | ------- | ------- | ------- | ------- | -------- |
| 1%   | 65.2%   | 850     | 58.3%   | 760     | 正T优先  |
| 2%   | 42.1%   | 550     | 36.7%   | 480     | 可做正T  |
| 3%   | 25.8%   | 336     | 20.5%   | 290     | 胜率偏低 |

输出：**最佳正T梯度**（胜率≥50%的最大梯度）和**均衡梯度**（正反T胜率都≥30%的最大梯度）。

**实战指导：**

- ATR/股价% > 3% 的股票适合高频做T（如科创板20%涨跌幅股票）
- 日内T触发条件：开盘后急跌至买入区间 → 买入 → 反弹至卖出区间 → 卖出
- 网格策略：每跌一个网格间距买一档，每涨一个网格间距卖一档，滚动降成本
- T梯度统计告诉你"历史上在X%振幅下做T胜率多少"

### 🎯 引擎 4：波段T策略（`analyze_swing_t()` + `analyze_current_swing_signal()`）

**回答的核心问题：** 在多日周期上，什么时候该买、什么时候该卖？

#### 4.1 波段做T历史统计

对每个周期（5/10/20/30/90日），统计历史区间涨跌幅达到各阈值时的后续走势：

| 维度     | 说明                                                              |
| -------- | ----------------------------------------------------------------- |
| 动态阈值 | 基于股票自身涨跌幅分布自动计算（非固定阈值），用百分位法取6个梯度 |
| 正T信号  | N日跌幅≥阈值 → 观察 N/2 天后是否反弹（买入信号）                |
| 反T信号  | N日涨幅≥阈值 → 观察 N/2 天后是否回调（卖出信号）                |
| 胜率统计 | 每个周期×每个阈值的触发次数、获胜次数、胜率、平均收益            |
| MACD分布 | 信号出现时 MACD 在什么方向（金叉/死叉/震荡）                      |
| 量能分析 | 放量/缩量/正常时的胜率差异（例如：缩量下跌后反弹概率更高）        |
| 背离分析 | 顶背离/底背离时的胜率差异（底背离+正T=高胜率组合）                |

#### 4.2 当前波段T信号（当日判定）

对指定日期（或日期范围），实时判断各周期是否触发买卖信号：

| 周期 | 涨跌幅 | 方向 | 触发阈值 | 信号类型 | 建议操作 | 历史胜率 |
| ---- | ------ | ---- | -------- | -------- | -------- | -------- |
| 5日  | -4.2%  | 下跌 | 3%       | 正T      | 买入     | 62.5%    |
| 10日 | -6.1%  | 下跌 | 5%       | 正T      | 买入     | 58.3%    |
| 20日 | +8.5%  | 上涨 | 5%       | 反T      | 卖出     | 55.1%    |

每条信号附带当时的技术指标：**MACD方向、RSI值、成交量状态、顶/底背离**。

#### 4.3 技术指标共振检验

| 共振级别 | 条件                                     | 含义           |
| -------- | ---------------------------------------- | -------------- |
| 强共振   | 多个周期信号方向一致 + MACD金叉/死叉确认 | 信号可靠性最高 |
| 中等共振 | 多个周期信号一致但MACD未确认             | 需谨慎         |
| 弱共振   | 信号分散，无一致方向                     | 建议观望       |

#### 4.4 综合决策评分

- 波段信号净得分（-100 ~ +100）：加权各周期信号（长周期权重更大）
- 结合 MACD/RSI/背离/极端程度 → 输出 **主策略 + 逻辑依据 + 置信度**

**实战指导：**

- 多个周期同时出现正T信号（如5日、10日、20日都超跌）→ 高概率反弹窗口
- 反T信号 + MACD顶背离 + RSI>70 = 强烈的卖出信号
- 正T信号 + MACD底背离 + RSI<30 = 强烈的买入信号
- 5日周期的信号最快但假信号多，90日周期信号最慢但最可靠

## 三、波段涨跌幅历史分布（`analyze_swing_amplitude_distribution()`）

**回答的核心问题：** 当前涨跌幅在历史上处于什么位置？是否极端？

对每个周期（5/10/20/30/90日），统计历史上 N 日区间涨跌幅的完整分布：

| 统计量                      | 说明                                             |
| --------------------------- | ------------------------------------------------ |
| 自适应分箱                  | 密集处区间窄、稀疏处区间宽，分布呈现更自然       |
| 上涨幅度分布                | 所有正收益的区间分布（含加权计数：越新权重越大） |
| 下跌幅度分布                | 所有负收益的区间分布                             |
| 均值/中位数/最大值/加权均值 | 常规统计量                                       |

**输出解读示例：**

```
若当前 20 日跌幅 -8.5%，历史 20 日跌幅分布：
  均值 -5.2%，中位数 -4.8%，最大值 -18.3%
  当前跌幅处于历史约 75% 分位（仅有 25% 的跌幅更深）
→ 属于较深回调，反弹概率较大
```

**实战指导：**

- 跌幅超过历史均值 1.5 倍 → 超跌信号，关注反弹
- 涨幅超过历史均值 1.5 倍 → 超涨信号，警惕回调
- 结合波段T信号使用效果更好

## 六、各分析维度的实战应用场景

### 场景 A：决定是否买入一只新股票

1. **趋势选时** — 看当前价格在均线系统的位置，MACD是否金叉
2. **估值定仓** — 看PE/PB历史分位是否在低位（<30%），ROE是否优秀
3. **波段T信号** — 看是否有多周期正T信号（超跌反弹机会）
4. **涨跌幅分位** — 看是否处于超跌区域（跌幅>历史均值1.5倍）
5. 综合判断：趋势偏多 + 估值偏低 + 多周期正T信号 = **高确定性买点**

### 场景 B：决定是否卖出手中的股票

1. **趋势选时** — 看均线是否转为空头排列，MACD是否死叉
2. **估值定仓** — 看PE/PB是否处于历史高位（>70%）
3. **波段T信号** — 看是否有多周期反T信号（超涨回调风险）
4. **涨跌幅分位** — 看是否处于超涨区域
5. 综合判断：趋势偏空 + 估值偏高 + 多周期反T信号 = **强烈的卖出信号**

### 场景 C：持仓股票如何降成本

1. **日内做T** — 根据ATR和T梯度统计，在开盘急跌时买入、反弹时卖出
2. **网格交易** — 按网格间距设置多档买卖单，自动低买高卖
3. **波段T** — 多日积累一定涨跌幅后执行波段买卖，降低平均持仓成本

### 场景 D：批量筛选多只股票

1. 对批量股票运行全部分析
2. 按 "趋势评分 + 估值评分" 排序
3. 找出 "趋势>60 + 估值>70 + 多周期正T信号" 的股票 → 优先买入
4. 找出 "趋势<40 + 估值<30 + 多周期反T信号" 的股票 → 优先卖出

## 八、配置文件自定义分析参数

```json
{
    "stocks": ["神火股份", "贵州茅台"],
    "start_date": "2019-01-01",
    "end_date": "2026-05-27",
    "t_start_date": "2026-01-01",
    "t_end_date": "2026-05-27",
    "analysis": {
        "grid_levels": 4,        // 网格交易档位数
        "atr_multiplier": 0.5,   // 网格间距 = ATR × 此系数
        "t_single_qty": 500      // 每笔做T股数，不设则自动估算
    }
}
```
