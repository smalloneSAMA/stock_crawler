# A股股票日K线数据抓取 & 智能分析工具

从配置文件中读取股票信息，通过 **新浪财经 API** 抓取日K线数据，结合 **东方财富** 财务数据填充基本面指标，
自动进行趋势选时、估值定仓、波动降本分析，并生成格式化的 Excel 文件。

## 🚀 功能概览

| 模块 | 功能 | 输出位置 |
|------|------|---------|
| 数据抓取 | 新浪财经日K线 + 东方财富财务指标(PE/PB/ROE/股息等) | `data/*.xlsx` |
| 智能缓存 | 已有数据本地读取，缺失部分才爬取更新 | `cache/*.cache.json` |
| 趋势选时 | 多周期均线/MACD/RSI/布林带综合评分 | `output/*_分析报告.txt` |
| 估值定仓 | 历史PE/PB百分位 + ROE质量评分 | `output/*_分析报告.txt` |
| 波动降本 | ATR日内T + 网格交易方案 | `output/*_分析报告.txt` |
| **日内做T分析** | 0.5%~10%梯度统计正T/反T胜率、平均收益 | `t_analysis/*_T策略分析.xlsx` |
| **波段做T分析** | 5/10/20/30日周期回测，含MACD/RSI/量能/背离 | `t_analysis/*_T策略分析.xlsx` |
| **波段T信号** | 对指定日期判断是否适合做T，结合技术指标 | `t_analysis/*_波段T信号_*.xlsx` |

## 🚀 快速开始

### 1️⃣ 安装依赖

```bash
cd stock_crawler
pip install -r requirements.txt
```

### 2️⃣ 修改配置

编辑 `config.json`，填入需要的股票信息：

```json
{
    "t_analysis_date": "2026-05-19",
    "stocks": [
        {
            "stock_name": "神火股份",
            "stock_code": "000933",
            "start_date": "2025-01-01",
            "end_date": "2026-05-19",
            "output_file": "神火股份抓取数据.xlsx",
            "analysis": {
                "grid_levels": 4,
                "atr_multiplier": 0.5,
                "t_single_qty": 500
            }
        }
    ]
}
```

#### 全局配置

| 字段 | 说明 | 是否必填 |
|------|------|---------|
| `t_analysis_date` | 波段T信号分析日期 (YYYY-MM-DD)，不填则跳过 | 可选 |

#### 股票配置

| 字段 | 说明 | 示例 |
|------|------|------|
| `stock_name` | 股票名称（仅展示用） | `"神火股份"` |
| `stock_code` | 股票代码（自动匹配沪/深市） | `"000933"` 或 `"000933.SZ"` |
| `start_date` | 数据起始日期 | `"2025-01-01"` |
| `end_date` | 数据截止日期 | `"2026-05-19"` |
| `output_file` | 输出Excel文件名 | `"神火股份抓取数据.xlsx"` |
| `analysis.grid_levels` | 网格交易档位数（可选） | `4` |
| `analysis.atr_multiplier` | 网格间距 = ATR × 此系数（可选） | `0.5` |
| `analysis.t_single_qty` | 每笔做T股数（可选，不设则自动估算） | `500` |

> **说明**：股票代码后缀 `.SZ`=深交所，`.SH`=上交所。不写后缀时 6/9 开头自动归沪市，其余归深市。

### 3️⃣ 运行程序

```bash
python main.py
```

也可指定自定义配置和输出目录：

```bash
python main.py -c my_config.json
python main.py -c my_config.json -o 输出目录
python main.py --no-cache              # 忽略本地缓存，强制重新抓取
```

## 🧠 做T策略逻辑说明

### 日内反T（先卖后买）

```
触发：开盘后价格上涨超过阈值%时，以 开盘×(1+阈值%) 挂卖单卖出
买回：收盘时以收盘价买回
盈利：收盘价 < 卖出价（价格冲高回落）
收益：实际收益率 = (卖出价-收盘价) / 卖出价 × 100%
```

### 日内正T（先买后卖）

```
触发：开盘后价格下跌超过阈值%时，以 开盘×(1-阈值%) 挂买单买入
卖出：收盘时以收盘价卖出
盈利：收盘价 > 买入价（价格探底回升）
收益：实际收益率 = (收盘价-买入价) / 买入价 × 100%
```

### 波段做T

```
对5/10/20/30日周期滑动窗口回测：
  - N日涨幅≥阈值 → 反T信号 → 观察未来N/2天是否下跌
  - N日跌幅≥阈值 → 正T信号 → 观察未来N/2天是否上涨
每个信号点附带MACD方向、RSI值、成交量状态、顶/底背离
```

## 💾 缓存机制

- 每次抓取完成后，数据自动缓存到 `cache/*.cache.json`
- 再次运行时检查缓存是否覆盖配置的时间范围
- 完整覆盖则跳过API抓取，缺失部分才增量更新
- 使用 `--no-cache` 参数强制重新抓取

```
第一次运行:  抓取API → 写入缓存
第二次运行:  检查缓存 → 日期完整 → 跳过API（秒级完成）
有新数据时:  检查缓存 → 日期不完整 → 仅补新数据 → 合并 → 更新缓存
```

## 📝 注意事项

1. **数据源**：K线来自新浪财经，财务数据来自东方财富数据中心
2. **复权方式**：使用不复权原始数据，涨跌幅按实际价格计算
3. **网络要求**：需要联网获取数据，API失败自动重试3次
4. **做T风险**：历史胜率不代表未来表现，未考虑交易佣金和印花税
5. **文件占用**：若Excel文件被打开，程序会自动另存为时间戳后缀文件
