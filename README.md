# 📈 A股股票日K线数据抓取 & 智能分析工具

**多源级联**抓取 A 股日K线数据，自动填充财务指标，进行 **趋势选时、估值定仓、波动降本、波段T信号** 四大分析，生成格式化 Excel + Markdown 决策报告。

---

## 🚀 数据源架构

| 优先级 | 数据源 | 协议 | 能力 | 特点 |
|--------|--------|------|------|------|
| 🥇 **mootdx** | 通达信 TCP | 日K线全量历史 | **分页获取**，可回溯至上市首日，不封IP |
| 🥈 **新浪财经** | HTTP | 日K线 ~1500条 | 原主力，JSON直连 |
| 🥉 **腾讯财经** | HTTP | 当日实时行情 | 保底，至少拿到当天OHLC |

> 📌 **关键优化**：mootdx 通过 `start/offset` 分页参数，突破单次800条限制，可获取股票全部历史数据（实测600519获取6300+条，从2001年至今）。

## 🚀 快速开始

### 1️⃣ 安装依赖

```bash
cd stock_crawler
pip install -r requirements.txt
```

> 如需生成 PNG 截图，还需下载 Playwright 的 Chromium：
> ```bash
> python -m playwright install chromium
> ```

### 2️⃣ 修改配置

支持 **JSON** 和 **Excel** 两种配置格式。

#### 格式一：JSON配置（`config.json`）

```json
{
    "stocks": ["神火股份", "贵州茅台"],
    "start_date": "2021-01-01",
    "end_date": "2026-05-27",
    "t_start_date": "2026-01-01",
    "t_end_date": "2026-05-27",
    "analysis": {
        "grid_levels": 4,
        "atr_multiplier": 0.5,
        "t_single_qty": 500
    }
}
```

| 字段 | 说明 | 是否必填 |
|------|------|---------|
| `stocks` | 股票名称数组或对象数组 | ✅ 必填 |
| `start_date` | 数据起始日期 | 可选，不限 |
| `end_date` | 数据截止日期（超今天自动修正） | 可选，不限 |
| `t_start_date` | 波段T分析起始日期 | 可选 |
| `t_end_date` | 波段T分析截止日期（超今天自动修正） | 可选 |
| `analysis.grid_levels` | 网格交易档位数 | 可选，默认4 |
| `analysis.atr_multiplier` | 网格间距=ATR×此系数 | 可选，默认0.5 |
| `analysis.t_single_qty` | 每笔做T股数 | 可选，自动估算 |

> **`t_start_date`/`t_end_date`** 替代旧的 `t_analysis_date` 字段，支持多日期自动生成交易日列表。

#### 格式二：Excel配置（`my_stocks.xlsx`，默认）

| stocks | start_date | end_date | t_start_date | t_end_date | enable |
|--------|-----------|---------|-------------|-----------|--------|
| 神火股份 | 2021-01-01 | 2026-05-27 | 2026-01-01 | 2026-05-27 | 1 |
| 贵州茅台 | 2020-01-01 | 2026-05-27 | | | 1 |

> 每行一只股票，各股票可独立设日期范围。`enable=0` 跳过该股票。

### 3️⃣ 运行程序

```bash
python main.py                           # 默认 my_stocks.xlsx
python main.py -c config.json            # 指定 JSON 配置
python main.py -c config.json -o 输出目录  # 自定义输出目录
python main.py --no-cache                 # 强制重新抓取
```

## 📊 数据范围建议

| 分析目标 | 建议年限 | 理由 |
|---------|---------|------|
| 🎯 **波段T信号分析（主要）** | **3~5年** | 覆盖盈亏周期 + 信号时效性好 + 样本充足 |
| 📈 趋势+估值+波段T综合分析 | **5~8年** | 与财务数据(30季报)对齐 + 多市场环境覆盖 |
| 💰 纯估值研究 | 全量历史 | PE/PB分位越多越准确 |
| 🆕 次新股(<3年) | 全量上市至今 | 数据本就不多 |

> **波段T信号**的 90 日周期每次消耗 135 个交易日，3 年数据提供约 345 个样本，5 年提供约 575 个。3 年为下限，5 年为最优。

### `output/*/买卖建议报告/*.md + .png` — 买卖建议报告

完整结构：
```
一、股票基础信息与估值分位 → 收盘价/PE/PB/市值历史百分位
二、价格与波段涨跌幅历史分位 → 各周期vs历史均值/中位数
三、当日综合策略信号(核心结论) → 正T/反T信号 + 历史胜率 + MACD/RSI/背离
四、辅助决策信息 → 日内做T Top3 + 技术指标共振检验
五、最终交易建议总结 → 主策略 + 买入区域 + 止损位 + 仓位建议
```

## 🧠 做T策略逻辑

### 日内反T（先卖后买）

```
触发：开盘后涨幅≥阈值% → 以 开盘×(1+阈值%) 卖出
买回：收盘价买回
盈利：收盘价 < 卖出价（冲高回落）
收益 = (卖出价-收盘价)/卖出价 × 100%
```

### 日内正T（先买后卖）

```
触发：开盘后跌幅≥阈值% → 以 开盘×(1-阈值%) 买入
卖出：收盘价卖出
盈利：收盘价 > 买入价（探底回升）
收益 = (收盘价-买入价)/买入价 × 100%
```

### 波段做T

```
对5/10/20/30/90日周期滑动窗口回测：
  N日涨幅≥阈值 → 反T(卖出)信号 → 观察N/2天后是否下跌
  N日跌幅≥阈值 → 正T(买入)信号 → 观察N/2天后是否上涨
每个信号点附带：MACD方向、RSI值、量能状态、顶/底背离
```

### 量能辅助

| 场景 | 规律 |
|------|------|
| **放量下跌+正T信号** | 恐慌盘涌出后反弹概率高 ✅ |
| **缩量下跌+正T信号** | 阴跌不止，抛压未竭 ⚠️ |
| **放量上涨+反T信号** | 冲高后获利盘涌出，易回落 ✅ |
| **缩量上涨+反T信号** | 无抛压，上涨可能持续 ⚠️ |

### 背离辅助

| 类型 | 含义 | 配合信号 | 效果 |
|------|------|---------|------|
| **底背离** | 价格新低但MACD未新低 | 正T(买入) | 提高胜率 |
| **顶背离** | 价格新高但MACD未新高 | 反T(卖出) | 提高胜率 |

## 💾 缓存机制

- 数据自动缓存到 `cache/*.cache.json`
- 再次运行时检查缓存是否覆盖日期范围
- 完整覆盖则跳过API，缺失部分增量更新
- `--no-cache` 强制重新抓取

```
首次运行:  mootdx分页获取全量 → 写入缓存 → 生成Excel
二次运行:  检查缓存 → 日期完整 → 跳过API（秒级完成）
有新数据:  检查缓存 → 缺最新日 → 仅补新 → 合并 → 更新缓存
```

## 📝 注意事项

1. **数据源**：K线来自 mootdx(通达信TCP) > 新浪财经 > 腾讯财经，三级自动降级
2. **复权**：使用不复权原始数据，涨跌幅按实际价格计算
3. **日期校验**：end_date/t_end_date 超过当天会自动修正为当天
4. **文件占用**：若Excel被打开，自动另存为时间戳后缀文件
5. **波段T**：建议 3~5 年数据，兼顾统计显著性和信号时效性
6. **风险提示**：历史胜率不代表未来表现，未考虑交易佣金和印花税
